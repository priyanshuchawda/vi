# Support

## Where To Ask For Help

- Use GitHub Issues for reproducible bugs and feature requests.
- Use Discussions (if enabled) or issue comments for implementation questions.
- For security vulnerabilities, do **not** open a public issue. Follow
  `SECURITY.md`.

## What To Include

- App version and platform (`macOS`, `Windows`, `Linux`).
- Repro steps (minimal and deterministic).
- Expected behavior vs actual behavior.
- Logs or screenshots when relevant.

## Response Expectations

- Maintainers will triage issues as time allows.
- Critical regressions and security issues are prioritized.
- Stale issues may be closed if there is no follow-up information.
